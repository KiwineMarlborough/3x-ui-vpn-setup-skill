# Инструкция: свой VPN за вечер с помощью ИИ

> **Если тебя достали блокировки чужих VPN-сервисов** — подними **личный** сервер. Этот skill учит ИИ-агента делать всё сам: от чистого VPS до рабочей подписки в Happ.
>
> **Твоя роль:** купить VPS, привязать домены, дать агенту SSH — и забрать готовые ссылки. **Не** нужно вручную править Xray, SQLite и nginx.

## Шаг 0. Что подготовить (15–30 минут, один раз)

| Что | Пример | Зачем |
|-----|--------|-------|
| VPS Ubuntu 22/24 | 1 GB RAM, DE/NL | Сервер VPN |
| IP адрес | `203.0.113.10` | A-записи DNS |
| Домен панели | `panel.mydomain.com` | Админка 3X-UI |
| Домен CDN | `cdn.mydomain.com` | VPN + подписка |
| SSH-ключ | `~/.ssh/vps_ed25519` | Вход без пароля |
| Sudo | пароль или NOPASSWD | Агент ставит пакеты |

**DNS:** две A-записи → IP VPS. В Cloudflare — **серое облако** (прокси выкл).

**Reality SNI:** любой чужой живой HTTPS-сайт, например `pixelforge.pics` — не свой VPN-домен. Skill проверит.

### Как безопасно дать sudo

Лучшие варианты (по убыванию):

1. **SSH-ключ** + sudo по ключу без пароля только на время setup — потом убрать из `/etc/sudoers.d/`
2. **Временный sudo-пароль** на 30 дней максимум — после handoff сменить: `passwd`
3. Отдельный пользователь `deploy` с sudo, не root — ключ в `authorized_keys`

Пароль root по SSH **не** давай. Панель 3X-UI агент создаст сам.

Подставь свои значения:

```text

  panel: panel.mydomain.com
  cdn:   cdn.mydomain.com

- scripts/verify-server.sh с SUB_ID
```

## Шаг 3. Что агент сделает сам

1. Обновит систему, UFW
2. Установит 3X-UI
3. Выпустит Let's Encrypt на оба домена
4. Поднимет nginx с «CDN»-страницей на 443
5. Создаст 4 inbound'а (Reality, TCP, XHTTP, Hysteria)
6. Настроит кастомные пути подписки
7. Включит Happ routing (.ru напрямую)
8. Прогонит `verify-server.sh`
9. Отдаст тебе **subscription URL** и параметры панели

---

## Шаг 4. Телефон (Happ Plus)

1. Скопировать subscription URL из handoff
2. Happ → добавить подписку
3. Потянуть вниз для обновления routing
4. Подключиться к профилю **Reality** (основной)

## Если что-то сломалось

**Не переустанавливай сервер сразу** — дай агенту repair-промпт:

```text
Skill 3x-ui-vpn-setup. Следуй references/repair-only.md.
Симптом: JSON подписка отдаёт HTTP 500.
Сначала audit-server.sh. Не переустанавливай.
```

| Симптом | Куда смотреть |
|---------|---------------|
| JSON sub 500 | Hysteria — `fix-hysteria-stream.py` |
| Sub 404 | Обрезанный sub_id или пути — `set-sub-paths.py` |
| Podkop не коннектит | `fix-podkop-flow.py` |
| Всё мертво | `audit-server.sh` → `repair-only.md` |

## FAQ

**Нужно ли уметь Linux?**  
Нет — если агент с SSH. Базово: купить VPS, прописать DNS, вставить промпт.

**Можно без домена?**  
Технически да, но skill рассчитан на домены + TLS + CDN fallback. С доменом стабильнее и незаметнее.

**Это легально?**  
Skill — инструмент настройки. Соблюдай законы своей страны и TOS хостинга.

**Агент увидит мои пароли?**  
Да, если вставишь в чат. Лучше: ключ SSH + временный sudo, после setup — ротация. См. `secrets-management.md`.
